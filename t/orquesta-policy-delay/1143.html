m4dcoder (W Chan) April 6, 2020, 9:03pm #6

There is no timeout specifically for workflow execution. It can sit in the queue until there is resource freed up to process it.

You are right, I forgot about the scheduler. There‚Äôs a scheduler sleep_interval (secs) option you can set in the st2.conf file. It defaults to 0.1 second. You can try to change it to an acceptable value that is easier on the system. Note that this will also delay normal scheduling up to the sleep interval.
